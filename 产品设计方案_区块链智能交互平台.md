# 基于 MCP 的区块链智能交互平台
## 产品设计方案（产品经历级）

---

## 一、项目背景

随着区块链生态的快速发展，链上数据与 DAO 治理信息呈现出爆炸式增长，但现有工具存在以下问题：

- 数据分散：链上数据、治理提案、市场数据分布在不同平台
- 信息理解成本高：用户难以快速理解 proposal 含义
- 缺乏智能交互：传统区块链浏览器仅提供检索，不提供分析能力

本项目基于 **Model Context Protocol（MCP）**，构建一个集：

> **链上数据获取 + DAO 治理信息 + 向量语义检索 + AI 智能交互**

于一体的智能平台。

---

## 二、产品定位

### 1. 产品名称
区块链智能交互平台（Blockchain Intelligent Interaction Platform）

### 2. 产品定位

> 面向 Web3 用户与研究者的 **智能区块链信息平台**

核心能力：

- 区块链数据聚合
- DAO 治理信息分析
- 向量语义检索
- AI 驱动的自然语言交互

### 3. 核心价值

| 传统工具 | 本平台 |
|--------|--------|
| 数据查询 | 数据 + 语义理解 |
| 手动分析 | AI 自动分析 |
| 信息分散 | 多源融合 |
| 无上下文 | MCP上下文感知 |

---

### 2. DAO 参与者
需求：

### 3. 区块链研究者

## 四、产品整体架构

```text
前端层（Dashboard / DAO / AI）
        ↓
API 层（统一服务接口）
        ↓
服务层
  - Market Service
  - Proposal Service
  - Vector Service
  - MCP Service
        ↓
数据层
  - MongoDB（原始数据）
  - Milvus（向量检索）
  - Kafka（数据流）
        ↓
数据来源
  - 区块链 RPC / API
  - Snapshot
  - Binance
```

## 五、核心功能模块设计

# 1️⃣ Dashboard（区块链总览模块）

## 1.1 模块目标

提供一个统一入口，展示：

- 主流币种信息
- 多链数据概览
- 实时价格信息

## 1.2 页面结构

### （1）全局搜索模块
功能：
- 搜索币种 / 链 / proposal

字段：
- 输入框
- 推荐关键词（ETH / BTC / Aave）

### （2）币种总览列表

展示字段：

| 字段 | 说明 |
|------|------|
| Symbol | 币种 |
| Name | 名称 |
| Chain | 所属链 |
| Price | 当前价格 |
| 更新时间 | 数据时间 |
| 操作 | 查看详情 |

### （3）系统状态面板

展示：

- 当前支持链数量
- 当前支持币种数量
- 已同步 proposal 数量
- 数据更新时间

## 1.3 币种详情页设计

模块划分：

### 基本信息
- Symbol
- 链
- 当前价格

### 链上数据
- 区块高度
- 交易信息（可扩展）

### 扩展入口
- AI 分析该币种
- 相关 DAO 信息

## 1.4 API 设计

### 获取总览
`GET /api/dashboard/overview`

### 获取详情
`GET /api/dashboard/detail`

### 全局搜索
`GET /api/search/global`

---

# 2️⃣ DAO 治理模块

## 2.1 模块目标

聚合主流 DAO 的 proposal，并提供：

- 浏览
- 查询
- 语义分析

## 2.2 支持 DAO（当前版本）

- Aave
- Uniswap
- ENS

## 2.3 页面结构

### （1）DAO 切换区

形式：
- Tab 切换

内容：
- Aave / Uniswap / ENS

### （2）Proposal 列表

展示字段：

| 字段 | 说明 |
|------|------|
| Title | 提案标题 |
| State | 状态 |
| Created | 创建时间 |
| Scores | 投票总量 |
| Keywords | 关键词 |
| 操作 | 查看详情 |

### （3）筛选与排序

功能：
- 状态筛选
- 时间排序
- 关键词搜索

## 2.4 Proposal 详情页

### 基本信息
- 标题
- 作者
- 状态
- 时间

### 正文信息
- Body
- Discussion

### 治理信息
- Choices
- Scores
- Total Score

### 语义信息
- 关键词
- 相似 proposal

### 操作
- 跳转 Snapshot
- AI 解读

## 2.5 API 设计

### Proposal 列表
`GET /api/dao/proposals`

### Proposal 详情
`GET /api/dao/proposal/detail`

### 相似检索
`GET /api/dao/proposal/similar`

### 最新 proposal
`GET /api/dao/latest`

---

# 3️⃣ AI 智能问答模块（核心模块）

## 3.1 模块目标

通过 MCP 实现：

> **自然语言 → 数据调用 → 智能回答**

## 3.2 页面结构

### （1）聊天主界面
- 输入框
- 对话记录
- 发送按钮

### （2）推荐问题区

示例：

- Aave 最近有哪些提案？
- ENS 最新治理变化是什么？
- ETH 当前价格是多少？

### （3）上下文面板（关键）

展示：

- 使用的 MCP 工具
- 命中 proposal
- 数据来源

## 3.3 交互流程

```text
用户输入问题
    ↓
MCP 解析意图
    ↓
调用对应工具
    ↓
返回数据
    ↓
AI 生成回答
```

## 3.4 MCP 工具设计

| Tool | 功能 |
|------|------|
| get_token_price | 获取价格 |
| get_wallet_assets | 获取资产 |
| search_proposals | 检索 proposal |
| get_proposal_detail | 获取详情 |

## 3.5 API 设计

### AI 聊天
`POST /api/ai/chat`

### 推荐问题
`GET /api/ai/suggestions`

### Proposal 解读
`POST /api/ai/explain-proposal`
