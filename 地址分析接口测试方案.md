# 地址分析接口测试方案

本文档用于测试 ChainPilot 新增的“地址分析 / Wallet Insight”模块。

## 1. 新增接口

### 1.1 获取支持链列表

```http
GET /api/wallet/chains
```

用途：获取地址分析模块当前支持的链。

### 1.2 地址资产分析

```http
POST /api/wallet/analyze
```

用途：输入钱包地址和链列表，返回该地址的多链资产画像、总资产估值、资产结构、风险提示和 DAO 治理关联信息。

请求体示例：

```json
{
  "address": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
  "chains": ["eth", "bsc", "polygon"]
}
```

## 2. 测试前准备

确认 `.env` 中配置了可用的 Ankr API：

```env
ANKR_API_URL=你的 Ankr Advanced API URL
```

启动后端服务：

```powershell
.\start.ps1
```

或直接使用 Uvicorn：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
```

默认后端地址：

```text
http://127.0.0.1:8000
```

## 3. 手动接口测试

### 3.1 测试支持链列表

PowerShell：

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/api/wallet/chains" `
  -Method GET
```

预期返回：

```json
{
  "chains": [
    {
      "key": "eth",
      "label": "Ethereum",
      "enabled": true
    },
    {
      "key": "bsc",
      "label": "BNB Smart Chain",
      "enabled": true
    },
    {
      "key": "polygon",
      "label": "Polygon",
      "enabled": true
    }
  ]
}
```

### 3.2 测试单链地址资产分析

使用 Vitalik 地址测试 Ethereum 链：

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/api/wallet/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{
    "address": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
    "chains": ["eth"]
  }'
```

预期检查点：

- `address` 等于请求地址
- `chains` 包含 `eth`
- `asset_count` 是数字
- `priced_count` 是数字
- `total_value_usdt` 是数字
- `risk_level` 是 `low`、`medium` 或 `high`
- `assets` 是数组
- `category_breakdown` 是数组
- `insights` 是数组

响应结构示例：

```json
{
  "address": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
  "chains": ["eth"],
  "asset_count": 3,
  "priced_count": 3,
  "total_value_usdt": 1234.56,
  "stablecoin_ratio": 0.12,
  "mainstream_ratio": 0.76,
  "defi_ratio": 0.08,
  "meme_ratio": 0.0,
  "governance_ratio": 0.08,
  "concentration_ratio": 0.7,
  "risk_level": "high",
  "risk_label": "较高",
  "assets": [],
  "category_breakdown": [],
  "governance_hints": [],
  "insights": []
}
```

### 3.3 测试多链地址资产分析

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/api/wallet/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{
    "address": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
    "chains": ["eth", "bsc", "polygon"]
  }'
```

预期检查点：

- `chains` 返回 `eth`、`bsc`、`polygon`
- `asset_count >= 0`
- `assets` 中可能出现不同 `blockchain`
- MongoDB 的 `asset_overviews` 集合新增记录

### 3.4 测试空 chains

如果 `chains` 为空，后端会默认使用配置中的全部支持链。

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/api/wallet/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{
    "address": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
    "chains": []
  }'
```

预期：

```json
{
  "chains": ["eth", "bsc", "polygon"]
}
```

## 4. 异常测试用例

### 4.1 不支持的链

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/api/wallet/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{
    "address": "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
    "chains": ["solana"]
  }'
```

预期状态码：

```text
400
```

预期返回：

```json
{
  "code": "WALLET_CHAIN_INVALID",
  "message": "No supported chain was selected."
}
```

### 4.2 空地址

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/api/wallet/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{
    "address": "",
    "chains": ["eth"]
  }'
```

预期状态码：

```text
400
```

预期返回：

```json
{
  "code": "WALLET_ADDRESS_REQUIRED",
  "message": "Field ''address'' is required."
}
```

### 4.3 格式明显错误的地址

```powershell
Invoke-RestMethod `
  -Uri "http://127.0.0.1:8000/api/wallet/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{
    "address": "not-a-wallet-address",
    "chains": ["eth"]
  }'
```

预期：

- 如果 Ankr API 返回错误，接口可能返回 `500`
- 后端日志会记录 Ankr API 调用失败
- 后续可考虑增加地址格式校验，将该场景优化为 `400`

## 5. 前端测试用例

访问：

```text
http://127.0.0.1:5173/wallet
```

测试步骤：

1. 页面顶部导航应出现“地址分析”。
2. 进入页面后应显示地址输入框和链选择按钮。
3. 默认链应包含 `Ethereum`、`BNB Smart Chain`、`Polygon`。
4. 输入测试地址后点击“开始分析”。
5. 分析成功后应展示：
   - 总资产估值
   - 资产数量
   - 风险等级
   - 稳定币占比
   - 主流资产占比
   - DeFi/治理占比
   - Meme 占比
   - 资产结构图
   - 资产明细列表
   - 治理关联卡片
6. 点击资产列表中的 Token，应跳转到 `/tokens/{symbol}`。
7. 点击“让 AI 继续分析”，应跳转到 AI 助手页面，并把地址画像内容填入输入框。

## 7. 推荐优先测试顺序

建议先测试这三个核心场景：

```text
1. GET /api/wallet/chains
2. POST /api/wallet/analyze，使用 Vitalik 地址 + eth
3. POST /api/wallet/analyze，使用 Vitalik 地址 + eth/bsc/polygon
```

如果这三个场景正常，说明后端接口、Ankr 数据源、Mongo 存储和前端页面主流程基本可用。
